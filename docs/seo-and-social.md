# SEO And Social

## Canonical Source

Profile SEO lives in `src/data/profile.ts`:

```ts
seo: {
  siteUrl: 'https://example.com',
  title: 'Name - Role',
  description: 'Specific one-sentence description.',
  keywords: ['keyword one', 'keyword two'],
}
```

Update `siteUrl` before publishing.

## Theme Colors

Site colors live in `src/data/profile.ts` under `theme`.

`src/layouts/LayoutBase.astro` applies those values as CSS variables:

```ts
theme: {
  paper: '#f7fbfd',
  surface: '#ffffff',
  accent: '#2f8193',
}
```

Tailwind utilities such as `bg-paper`, `bg-surface`, `bg-accent`, and `border-line` resolve through those variables, so a profile can change the primary color and background tone in one place.

## Metadata

`src/layouts/LayoutBase.astro` emits:

- title
- description
- canonical URL
- Open Graph tags
- Twitter card tags
- RSS discovery
- `llms.txt` discovery
- basic AI crawler permissions
- JSON-LD structured data

## Open Graph Image

`src/pages/og.png.ts` generates a 1200 by 630 social image from `src/data/profile.ts`.

Run the build after editing profile text. Long names and summaries should still fit inside the generated image.

## Structured Data

Homepage schema:

- `ProfilePage`
- nested `Person`
- `sameAs` links from public social links

Post schema:

- `Article`
- author set to the profile name
- exact published and modified dates

## RSS And Sitemap

- RSS route: `/rss.xml`
- Sitemap: generated by `@astrojs/sitemap`
- Update `public/robots.txt` with the production domain.
